4      Approaches

 

4.1    Monte Carlo Ray Tracing

The rendering equation can be solved using Monte Carlo techniques. To do so, a high number of directional samples are produced, the rendering function is evaluated for every sample and the average of all results converges to the true solution. To evaluate a sample, the incoming light from one direction has to be computed. This is usually done using ray tracing: a ray is sent in the direction and the light emitted from the first hit point is computed, potentially, again by computing a solution of the rendering equation at this location.

4.1.1      Final Gathering

The final gathering method reconstructs the incoming radiance values over the hemisphere for each point visible though the pixel read though through the stored radiance solution. Radiance at each point can be estimated using photon density. Final gathering aims to reduce photon artifacts.

4.1.2      Bidirectional Tracing

Bidirectional ray tracing generates paths starting at the light sources and the surface point simultaneously and connects both paths in the middle to find a contribution to the light transport between the light source and the point for which a radiance value needs to be computed.

The algorithm has the availability of two different path generators when computing a Monte Carlo estimate for the flux through a certain pixel.

4.1.3      Metropolis Light Transport

Metropolis light transport is a stochastic ray tracing technique where the overall strategy is to sample paths with probability proportional to the image contribution function. To do this, we generate a sequence of paths  where each patch is obtained by a random mutation to the path . The mutations can have almost any desired form, and typically involve adding, deleting or replacing a small number of vertices on the current path.

The metropolis framework gives a recipe for determining the acceptance probability for each mutation such that in the limit the sampled paths  are distributed according to the image contribution function.

4.2    Finite Elements

The underlying idea is to discretize the scene’s surfaces into finite surface elements, often called patches, and then compute the light transport between them.

For every patch one need to store the radiosity value for diffuse surfaces, or the directional distribution of incoming and outgoing light in the non-diffuse case, the light transport amounts to solving a linear system of equations once the form factors, which denote the amount of light transport between two patches, are known.

4.2.1      Radiosity on graphics hardware (2004)

Here is presented a technique for computing radiosity, including an adaptive subdivision of the model. The implementation that is equivalent to conventional patch orientated radiosity on a uniformly subdivided domain. This system stores the radiosity and the residual as textures on the modeling polygons. The scene polygon with the highest total residual energy is chosen as the next shooter. We use the hemicube method to sample visibility from the current shooter. We divide the shooter into m subsections and proximate their areas with oriented discs.

If an element is visible from the shooter, then the next step is to compute the intensity that it receives. We use the disc approximation to a differential area-to-area form factor equation between two elements I and j.

This method subdivides the scene geometry hierarchically into a quadtree with the radiosity data stored at the leaf nodes.


To render a texture quadtree, we traverse it recursively. When a leaf node is reached, a quadrilateral is rendered with the node’s radiosity texture. We display all scene polygons textured with the adaptive radiosity solution.

4.2.2      Dynamic Ambient Occlusion and Indirect Lighting (2005)

This paper introduces a new technique for computing diffuse light transport and show how it can be used to compute global illumination for animates scenes. This technique works by treating polygon meshes as a set of surface elements that can emit, transmit or reflect light and that can shadow each other.

We define polygon meshes as disk-shaped elements, one element created for each vertex. Elements defined by position, normal and area.

Ambient occlusion provides soft shadows by darkening surfaces that are partially visible to the environment. It involves calculating the accessibility value, which is the percentage of the hemisphere above each surface point not occluded by geometry. We also calculate the direction of least occlusion (bent normal).

We calculate the accessibility values for each vertex in a multipass algorithm that removes multiple shadowing effects.

We compute the bent normal by first multiplying the normalized vector between elemenets and the form factor. Then we subtract this result from the original element.

We calculate the occlusion result be rendering the result to a texture. The shader calculates the amount of shadow received at each element and writes it to the alpha component of the pixel. Bent normal is also rendered. We calculate the amount of light transferred from one surface element to another using the geometric disk-to-disk form factor. We run one pass of this for our radiance-transfer shader to calculate the max amount of reflected or emitted light that can reach any element. Then we run a shadow pass that subtracts from the total light at each element based on how much light reaches the shadowing elements.

4.2.3      Voxel-based global illumination (2011)

This paper introduces Voxel-based Global Illumination (VGI), a scalable technique that ranges from real-time near-field illumination to interactive global illumination solutions. It introduces a new atlas-based boundary voxelization algorithm and an extension to a fast ray-voxel intersection test.

We render the complete object to a single atlas texture image in one rendering pass. World-space positions have to be written to the atlas and during that process arbitrary other data can be stored as well. All valid entries in the texture atlas are inserted into the grid storing the voxel data using point rendering. The voxelization camera’s frustum defines the region that is voxelized. A vertex is generated for each valid texel of the atlas texture.

4.3.2      Interactive global illumination in dynamic scenes

This paper presents a system for interactive computation of global illumination in dynamic scenes. Our system uses a novel scheme for caching the results of a high quality pixel-based renderer such as bidirectional path tracer.

They have designed a novel scheme for caching high quality samples generated by pixel-based rendering algorithms such as path tracing and then interpolating between these samples using graphics hardware. The shading cache is a hierarchical subdivision mesh attached to each primitive in the scene, with lazily evaluated shading values at the mesh vertices. It is refined by choosing locations in the image plane that require more accuracy. The selection of these locations is made using a priority map, which capture the likelihood of error due to interpolation, view-dependence and object motion.

This paper also introduces a novel flood filling operator for efficiently capturing the high frequency detail in the images to supplement the priority based sampling. Finally the graphics hardware is used to efficiently interpolate between the shading samples.

During initialization, the mesh for each geometric primitive is set to be a single un-subdivided patch. The shading values for these patches are not pre-computed, but evaluated lazily when required for generating a good image. When a patch is selected for update, we either evaluate the shading values for its vertices or subdivide the patch into four children and evaluate the shading values for their vertices.

4.3.4      The RADIANCE light simulation and rendering system

This paper describes a physically-based rendering system tailored to demands of lighting design and architecture. The simulation uses a light-backwards ray-tracing method with extensions to efficiently solve the rendering equation under most conditions.

Radiance uses a ray-tracing in recursive evaluation of the rendering equation. Indirect lighting changes gradually over surfaces. It should be possible to spread out this influence over many pixels to obtain a result that is smooth and accurate at a modest sampling cost. The original method for computing and using cached irradiance values has been enhanced using gradient information.

The basic idea is to perform a full evaluation of the rendering equation for indirect diffuse contributions only as needed, caching and interpolating these values over each surface. We are free to adjust the density of out calculation point in response to the illumination environment.

To compute the indirect irradiance at a point in our scene, we send a few hundred rays that are uniformly distributed over the projected hemisphere. If any of our rays hits a light source we disregard it since the direct contribution is computed separately. The sampling process is applied recursively for multiple reflections. We have developed a numerical approximation to the irradiance gradient based on hemisphere samples. Knowing the gradient in addition to the value of a function, we can use a higher order interpolation method.

We reduced the number of shadow rays with a prioritized list of potential source contributions at each evaluation of the rendering equation. The largest potential contributions are tested for shadows first, and we stop testing when the remainder of the source list is below some fraction of unoccluded contributions.

4.4    Photon Mapping

Photon Mapping is a two pass global illumination method.

The photon maps are constructed by emitting a large number of photons from the light sources in the scene. Each photon is traced through the scene using a method similar to path tracing. Every time a photon hits a surface it is stored within the photon map and Russian roulette is used to determine whether the photon is absorbed or reflected. The new direction of a reflected photon is computed using the BRDF of the surface.

We use two photon maps:
Caustic photon map: The caustic photon map is used only to store photons corresponding to caustics and is created by emitting photons towards the specular objects in the scene and storing these as they hit diffuse surfaces. Caustics are rendered by visualizing a radiance estimate based on the caustics photon map directly.

Global Photon Map: The global photon map is used as a rough approximation of the light/flux within the scene and is created by emitting photons towards all objects. It is not visualized directly and therefore it does not require the same precision as the caustic photon map.

The photons are stored in a balanced kd-tree.

To compute the radiance , leaving an intersection point x at a surface with BRDF , we locate the N photons with the shortest distance to x. Based on the assumption that each photon p represents flux  arriving at x from direction  we can integrate the information into the rendering equation.

Caustics are evaluated using the information in the caustics photon map. The radiance estimated based on the caustics photon map is visualized directly and this is the reason why the number of photons in the caustics photon map must be high.

Soft indirect illumination is incoming light which has been reflected diffusely at least once since it left the light source. This light is then reflected diffusely by the surface and consequently the resulting illumination is very “soft”. The approximate evaluation of this integral is the radiance estimate based on the global photon map.

4.4.1      Progressive Photon Mapping (2008)

The main idea in progressive photon mapping is to reorganize the standard photon mapping algorithm based on the conditions of consistency, in order to compute a global illumination solution with arbitrary accuracy without storing the full photon map in memory.

Progressive photon mapping is a multi-pass algorithm in which the first pass is ray tracing and all subsequent passes use photon tracing.

Ray Tracing Pass:
Use standard ray tracing to find all the surfaces in the scene visible through each pixel in the image. Each ray path includes all specular bounces until the first non-specular surface seen. In scenes with large number of specular surfaces the length of ray paths can be limited by Russian roulette.

For each ray path we store all hit points along the path where the surface has a non-specular component in the BRDF. We store data points in the struct hitpoint.

Photon Tracing Passes:
The photon tracing step is used to accumulate photon power at the hit points found in the ray tracing pass. It can be divided into multiple passes where each pass consists of tracing a given number of photons into the scene in order to build a photon map.

After each photon tracing pass we loop through all hit points (from the ray tracing pass) and find the photons within the radius of each hit point. We use the newly added photons to refine the estimate of the illumination.

Once the contribution of the photons have been recorded they are no longer needed, and we can discard all photons and proceed with a new photon tracing pass. This continues until enough photons have been accumulated and the final image quality is sufficient.

4.4.2      Simulating Photon Mapping for Real-time Applications (2004)

This paper introduces a method for simulating photon mapping. It exploits the main idea behind photon mapping namely that the different elements in global illumination can be calculated separately (direct, specular, caustics and indirect)

Photons are intelligently selected to be re-emitted. Photons from light sources are grouped together where each photon in each group has similar starting directions. Each photon is enumerated in each group. One photon is emitted from each group every frame. The path of each photon is stored, and if the new path diverges from the previous path, all photons from the group will be marked for redistribution.

The photons are stored as photon-hits on the surfaces along the path of the traced photon. The complete path of each photon is also stored. Each surface also has pointers to photon-hits that have hit that particular surface.

To approximate global illumination we compute the irradiance using N-nearest neighbor query of the photons. The radiance of the surface is stored in reconstruction texture maps applied to the surface.

4.4.3      Low latency photon mapping using block hashing (2002)

Typical KNN photon mapping algorithm use spatial partitioning. Here, we present a neighborhood-preserving hashing algorithm that is low-latency and has sub-linear access time.

Our algorithm uses hash functions to categorise photons by their positions. Then a kNN query proceeds by deciding which hash bucket is matched to the query point and retrieving the photons contained inside the hash bucket for analysis.

4.4.4      Interactive global illumination using selective photon tracing (2002)

This method exploits temporal coherence of illumination by tracing photons selectively to the scene regions that require illumination update.

Photon tracing is based on Quasi-Monte Carlo sampling with the multi-dimensional Halton sequence. The main problem is to identify each invalid photon and to replace it by the corresponding photon, which is traced for the current scene configuration. We show how the periodicity property of this sequence, allows for immediate identification of photons that have similar paths in the scene. Thus, for any invalid photon, it is easy to identify indices of all similar photons in Z that are likely to be invalid as well.

4.4.5      Photon mapping of programmable graphics hardware (2003)

This paper presents a modified photon mapping algorithms capable of running on GPUs. The implementation uses photon tracing to distribute the photons on the GPU. Photons are stored in a grid based photon map, constructed on the GPU. A KNN method is also described.

4.4.6      Interactive screen-space accurate photon tracing on GPUs (2006)

This paper presents a new method to directly render caustics on complex objects. The algorithm is based on the rasterization of photon paths in texture maps. Intersection events are then resolved on a per-fragment basis using layered depth images.

This method provides an effective means to resolve inter-object light transfer as well as several refractions at complex and dynamic polygonal objects.

Photon rays are traced in screen-space on programmable graphics hardware. This is done by rendering line primitives with respect to the current view. To be able to resolve intersection events at arbitrary position along these lines they are rasterized into texture maps. Intersections between photon rays and objects in the scene can now be detected using layered depth maps and simple fragment operations. By rendering oriented point sprites at the receiver pixels in screen-space we account for the energy transport through beams of light. The use of energy splats significantly minimizes the number of photons that have to be traced.

4.4.7      Real-time KD-tree construction on graphics hardware (2008)

The paper presents an algorithms for constructing kd-trees on GPUs. Specifically the algorithm builds tree nodes in breadth-first search order to fully exploit the fine-grained parallelism of modern GPUs at all stages of kd-tree construction.

4.4.8      An efficient GPU-based approach for interactive global illumination (2009)

This paper presents a GPU-based method for interactive global illumination that integrates complex effects such as multi-bounce indirect lighting, glossy reflections, caustics, and arbitrary specular paths. Our method builds upon scattered data sampling and interpolation on the GPU.

It starts with raytraced shading points and partitions them into coherent shading clusters using adaptive seeding followed by k-means. At each cluster center we apply final gather to evaluate its incident irradiance using GPU-based photon mapping. We approximate the entire photon tree as a compact illumination cut, this reducing the final gather cost for each ray. The sampled irradiance values are then interpolated at all shading points to produced rendering. Our method exploits the spatial coherence of illumination to reduce sampling cost. We sample sparsely and the distribution of sample points conforms with the underlying illumination changes. Therefore our method is both fast and preserves high rendering quality.

 

 

4.4.9      Hardware-accelerated global illumination by image space photon mapping (2009)

ISPM is a screen-space algorithm which takes two steps of the photon mapping algorithm, the initial bounce and the photon gathering. The initial bounce is calculated for each light and stored in a bounce map. Photon gathering is done by splatting the final photon volumes to produce indirect lighting.

The first step of ISPM is the render light GBuffers – a GBuffer for each light. Each Light GBuffer describes the different BSDF properties of the surfaces at each pixel in light space.

The initial photon bounce is made by rasterizing a bounce map for each emitter in the scene. A fragment shader is used to emit a photon through each pixel center. The pixels are projected onto a sphere around the emitter. When the photon is emitted, the first bounce is calculated using the light GBuffers. The GBuffer contain the necessary information for where the photon will hit the surface and the properties of the surface.

Overview of the algorithm:

·         For each light: render the light GBuffers and render the light bounce map.

·         Render the direct lighting.

·         Render subsampled eye GBuffer.

·         Extract and trace the photon from the bounce map.

·         Render the indirect lighting by splatting the photons.

·         Upsample the indirect lighting and combine it with the direct lighting.

·         Add color correction, post effects and display the image.

4.4.10   Interactive global photon mapping (2009)

This paper presents a photon mapping technique capable of computing high quality global illumination at interactive frame rates. By extending the concept of photon differentials to efficiently handle diffuse reflections, we generate footprints at all photon hit points.

The final image is computed by ray tracing with indirect illumination retrieved from the photon map. By combining these with a highly parallel BVH build and density estimation based on photon footprints.

4.4.11   Multi-image based photon tracing for interactive global illumination of dynamic scenes (2010)

In this paper the image space approach is extended by using multiple environment maps for photon mapping computation to achieve interactive global illumination of dynamic complex scenes.

4.4.12   Parallel progressive photon mapping on GPUs (2010)

The key contribution is a new stochastic spatial hashing scheme that achieves a data-parallel construction of a photon map and an efficient range query of a photon map on GPUs. This method provides a very simple way to map computation of progressive photon mapping into data parallel computation. The new hashing scheme achieves about the same performance as the state of the art method in this implementation, but with a significantly simpler algorithm.

In a typical CPU implementation of spatial hashing, each hash entry keeps a list of elements when a hashing collision occurs. This solution is to stochastically store a single photon instead of storing a list of photons at each hash entry. More precisely, given n photons that would be mapped to the same hash entry, we select on photon according to the uniform probability p(x).

4.5    Instant Radiosity

Instant Radiosity is a separate class of algorithms similar to photon mapping but is based on gathering rather than density estimation. Instant Radiosity is a two pass technique: In the first pass, photons are emitted and bounced inside the scene. The second pass, each photon is treated as a virtual point light (VPL) that emits light into the scene.

4.5.1      Instant Radiosity (1997)

Radiance is calculated using a discrete density of virtual light sources. Radiance is calculated using quasi-Monte Carlo integration and photon transport is done using quasi-random walk.

4.5.2      Bidirectional Instant Radiosity (2006)

The goal of bidirectional instant radiosity is to find a relevant set of virtual light sources (VPL’s). Instead of generating VPL’s only from light sources, they are also generated from the camera. The end of such path gives the location of a VPL which can bring light to the camera after one bounce.

4.5.3      Reflective Shadow Maps (2005)

A reflective shadow map is an extension of a standard shadow map, where every pixel is considered an incident light source. We render the scene from the perspective of the light source. In a reflective shadow map, with every pixel, we additionally store the light reflected off the hit surface.

4.5.4      Incremental Instant Radiosity for Real-Time Indirect Illumination (2007)

This method is a variant on instant radiosity. In this method hardware shadow mapping is employed for determining visibility between the VPLs and the visible surfaces of the scene. Another method is introduced of reusing VPLs so that their shadow maps need not be computed every frame. After training the neural network, the indirect illumination of each scene point can be rendered under different lighting and view conditions.
